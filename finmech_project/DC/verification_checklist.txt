**Consent to Check:**
•	**Do not** count **"Haan"**, **"Haan ji"**, **"Ok"**, **"Hmm"** as consent. First apply **Proactive vs Passive** and **Case 1 / Case 2 (Smart Sale)** below.
•	**Informal but substantive consent** (only if pattern is proactive): Phrases that **authorize** the upgrade, e.g. *"Ok, request le lijiye"*, *"Theek hai, kar dijiye"*, *"Yes"*, *"Ok, proceed"* **when** they clearly answer the **permission question**, not mid-pitch *Theek hai?* check-ins.

•	**Always invalid:** **"Hann / Haan / Hann ji"** or **"Ji"** **alone** as the only response to the **final** consent request in a call where the customer was **not** proactively engaged — **Consent = NO** 

•	Confused Consent: Cases where initial consent was denied and reconfirmed later are valid, Mark Consent Taken = YES, unless the customer not denying 2 times clearly or 1 ti,e at the end of call transcript. 

•	Consent Taken = NO when the Agent didn’t summarize the details before ending the call or while taking the Consent. This should be Proper without any mistake.

•	Language-Specific Consent: Regional languages (e.g., Marathi, Telugu, Kanada, Tamil) where standard words may differ.

##**Remember (strict)**
o	After taking consent if further customer was saying something at the end and call suddenly disconnected or if conversation seems incomplete then also Mark Consent = NO.
o	Mark Consent Taken = NO, if the customer clearly refuses or denies to the upgrade two or more times and agent forced repeatedly and taken the consent or one time deny at the end. 
o	Focus on whether the customer agreed or not after the attempt to convince. But not pushed more than 2 times after customer denying consistently, if this happens Mark Consent Taken = NO.
o	If the customer hesitates or denied to upgrade the card even after the benefits and charges are explained. Mark Consent Taken = NO.

o	If the customer clearly disagreed with the agent or refusing and showing any concern to upgrade the card even after the all the details are explained then Mark Consent Taken = NO. 

**Strict Evaluation Rules with Exception**
•	If Customer denied initially, initial deny is common, it should not be treated as No.
•	If customer raises concerns about charges and agent does not clarify properly before taking consent, it is invalid consent and marked as NO.
Smart Sale Indicators (Smart Violation)

**A call must be identified Smart Sale / Misleading if its aligns to below:**
•	**Reference pattern 1 (invalid):** Agent explains benefits/charges; customer only **“Ok”** after a statement block; agent asks *kya request le sakte hain*; customer again **“Ok”** only; agent summarizes; same **“Ok”** — **Rework.** 
•	**Reference pattern 2 (invalid):** Agent self-dialogue and rapid pitch; customer only **“Haan ji”** in gaps; call **cuts before** final summary/complete consent → **Consent = NO** + **incomplete call**; **Rework.**

**Note:** If all the time customer saying "haan" or "ok" then the customer is seems not aware of this upgrade process and this should not be Approved.

##**Mark Consent = Yes, if:**
1.	Customer gives one of the acceptable explicit responses with clear understanding after Agent asking for Consent for request to upgrade Card.
 
2.	Customer Clearly understand the details and agrees. Also, should not left any doubt on card upgrade before the consent.

3.	And If agent explain correctly on every query of customer. Also responding exactly and accurately without manipulating or ignoring the customer’s question.

1️⃣ Consent was unclear, **incomplete**, or **acknowledgment-only** (standalone **"Ok"** / **"Okay"** / **"Haan ji"** / **"Hmm"** in a **non-proactive** call)
2️⃣ **Incomplete call / incomplete summary** before final consent (Scenario 8, 13, Q12)
3️⃣ If Consent taken from Third party.
4️⃣ Charges not fully disclosed
5️⃣ Forced lead attempt.
6️⃣ If Customer seems not understanding and confused.
7️⃣ **Passive consent:** no **proactive** customer participation; consent is just **"OK"**, **"Haan"**, or **"Haan ji"** as **acknowledgment**, not as **clear authorization** of the upgrade request
8️⃣

Recommended Evaluation Checklist (Quick QA Version)
#**Evaluators should ask these 10 strict questions:**
1.	Was **substantive, proactive** consent obtained — **not** only **Ok / Haan ji / Hmm** as acknowledgments in a **non-proactive** call? (Must be YES)
2.	Did the agent avoid manipulating the consent to convert the call into a lead? (Must be Yes)
3.	Were both charges disclosed? (Must be YES)
4.	Was lounge condition explained correctly if explained? (Must be YES)
5.	Was lead pushed too much / forced? (Must be NO)
6.	Was the customer denies the card multiple times? (Must be NO)
7.	Did the agent ask a clear upgrade consent question before proceeding? (Must be YES)
8.	Did the agent allow the customer to fully respond before proceeding with consent capture? (Must be YES)

If all the above questions passed then Mark Call_Status = Approved, else Mark Call_Status = Rework.

Must provide all the reason for either Approving or Reworking the call Staus.
